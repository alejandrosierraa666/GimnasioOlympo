# Proyecto Colaborativo UT5

## Descripción

Este repositorio contiene el **Proyecto Colaborativo de la UT5: Utilización de técnicas de acceso a datos**. El objetivo del proyecto es desarrollar una **aplicación web colaborativa** cumpliendo una serie de requisitos técnicos, funcionales y organizativos, haciendo uso de **PHP, MySQL y GitHub**.

El proyecto se realiza en **grupos de 2 personas**, trabajando de forma conjunta mediante control de versiones.

---

## Requisitos del proyecto

* Uso de **GitHub** como repositorio colaborativo.

  * Todos los miembros del grupo pueden modificar el código fuente.
  * El profesor tiene acceso como colaborador.
  * Se realizan subidas de código **al menos dos días a la semana**.

* La aplicación web final será validada mediante métricas como:

  * Velocidad
  * Accesibilidad
  * Buenas prácticas

* Elaboración de un **manual básico de usuario** que explique el funcionamiento de la aplicación.

---

## Requisitos de la aplicación

### Autenticación y roles

* Sistema de **login y registro de usuarios**.
* Solo los usuarios registrados pueden acceder a la aplicación.
* Implementación de **al menos 2 roles de usuario**.

  * Dependiendo del rol, el usuario verá diferentes controles e información.

### Base de datos y CRUD

* Uso de una base de datos relacional **MySQL**.
* La base de datos contiene **al menos 3 tablas**.
* Creación de la base de datos y tablas mediante **código PHP**.
* Implementación completa de operaciones **CRUD**:

  * Create
  * Read
  * Update
  * Delete
* Uso de **consultas preparadas (prepare)** al menos en una consulta.

### Sesiones y cookies

* Uso de **al menos 2 cookies**.
* La sesión del usuario expira tras un tiempo de **inactividad**.

### Estructura y buenas prácticas

* Código correctamente estructurado mediante:

  * Funciones
  * Excepciones
  * Estructuras de control

* Creación de un **log en el servidor** que registre:

  * Mensajes de error
  * Mensajes informativos

### Funcionalidades adicionales

* Envío de **correos electrónicos** mediante un formulario de contacto.

## Manual de usuario (resumen)

1. Registrarse en la aplicación.
2. Iniciar sesión con usuario y contraseña.
3. Acceder a las funcionalidades según el rol asignado.
4. Realizar operaciones CRUD disponibles.
5. Usar el formulario de contacto para enviar correos.
6. Cerrar sesión o esperar la expiración automática por inactividad.
